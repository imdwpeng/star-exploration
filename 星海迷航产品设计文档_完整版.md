# 1. 产品概述

## 1.1 产品愿景

打造一款融合"三维消除"、"叙事探索"与"轻度社交"的下一代休闲手游，为玩家提供沉浸式的宇宙探险体验。

## 1.2 核心定位

目标用户：18-35岁休闲手游玩家，科幻爱好者，喜欢《羊了个羊》但渴望更深层次体验的用户
平台：iOS/Android/微信小程序（轻量版）/PC（Steam）
核心循环：消除→收集→建设→探索→社交分享

## 1.3 关键指标（KPI）

• 次日留存率：>40%
• 7日留存率：>20%
• 平均会话时长：>12分钟
• 付费转化率：3-5%
• 用户生成内容（UGC）关卡月活跃创作者：>1万人

# 2. 核心玩法设计

## 2.1 核心循环与唯一目标

单局游戏目标：清空三维星图上的所有方块。
失败条件：屏幕底部的"待消除槽"被填满，或者倒计时为0。

## 2.2 基础操作与核心规则（重中之重，必须明确）

规则1：消除发生的唯一地点
消除只发生在屏幕底部的待消除槽中。游戏场（三维星图）上的方块永远不会因为彼此相邻而自动消除。

规则2：消除触发的唯一方式
玩家必须手动点击场上的一个方块，将其移入待消除槽。当且仅当待消除槽内因这次放入而存在至少3个相同类型的方块时，这组方块会立即从槽中消除，并为玩家补充基础能量（防止失败的关键资源），增加相应倒计时。

规则3：场上图案与资源奖励的独立判断
"场上方块的排列图案"是一个独立的奖励系统，与"槽内能否消除"无必然关系。
每当玩家点击选择一个场上方块时，系统会同步进行两次独立判断：

判断1: 槽内兼容性判断：这个方块放入后，待消除槽内是否有3个同类的？

判断2: 场上奖励判断：这个方块在被移走前，在三维星图中是否构成了某个特定的"奖励图案"的一部分？

只有"判断1"直接影响消除和游戏进程。"判断2"只影响额外奖励资源的获取。

场景：场上有三个红色方块水平相邻（构成奖励图案"晶簇"）。待消除槽中已有2个红色方块。

## 2.4 外围成长系统

外围成长系统包括太空站建设和科技树系统，为玩家提供长期目标和个性化发展方向。

# 3. 游戏模式

## 3.1 主线模式：星海编年史

8个主题星系，每个星系5-8个关卡。每个星系有独特剧情、美术风格和专属机制。

## 3.2 合作探索（异步+实时）

异步合作：好友可派遣无人机构造你的太空站。
实时协作：3人小队挑战"深渊裂隙"特殊关卡，玩家各自负责一个维度的消除，共享资源池，共同决策建设选择。

## 3.3 创意工坊（UGC）

提供关卡编辑器，玩家可自定义方块初始布局、胜利/失败条件、自定义事件规则。发布后接受社区评分，高分关卡进入官方推荐。

## 3.4 赛季与活动

每季度一个主题赛季（如"虫族入侵"），赛季通行证（免费+付费），限时特殊关卡和排行榜。

# 4. 技术架构（微信小游戏专项版）

## 4.1 引擎与平台选型

• 核心引擎：Three.js r160
• 目标平台：微信小游戏（主平台），兼顾iOS/Android原生应用（后续扩展）
• 开发环境：VS Code + 微信开发者工具
• 构建目标：微信小游戏（主平台），兼顾iOS/Android原生应用（后续扩展）
• 渲染后端：WebGL 2.0（Three.js原生支持）

## 4.2 微信小游戏关键技术限制与应对

## 4.3 关键算法与性能优化

• 三维布局生成算法：保证每关有至少3种可行解，算法复杂度O(n²)
• 动态资源加载算法：基于玩家进度预测加载CDN资源，减少等待时间
• 内存优化算法：自动检测并释放长时间未使用的资源，LRU缓存策略
• 启动优化：使用Webpack代码压缩和资源优化，首包体积减少60%
• 渲染优化：使用Three.js InstancedMesh批量渲染同类方块，减少Draw Call，目标移动端60fps
• 场景管理：使用Three.js Scene/Group层级管理，按需加载/卸载场景节点
• 动画优化：使用requestAnimationFrame驱动渲染循环，避免不必要的重绘

## 4.4 数据存储架构

• 玩家进度：本地加密存储（微信文件系统）+ 云端备份（可选腾讯云数据库）
• 游戏配置：远程JSON配置，支持热更新，无需重新发布小游戏
• UGC关卡：分布式存储（腾讯云COS）+ CDN加速，全球玩家快速访问
• 防作弊：关键操作服务器验证，本地数据签名校验，防止修改器
• 数据分析：集成微信数据助手，实时监控用户行为与付费转化

## 4.5 开发与发布流程

1. Three.js开发阶段：使用Three.js引擎，针对微信小游戏平台开发，注意避免使用微信不支持的API
2. 本地测试：VS Code + 微信开发者工具模拟器，验证基础功能
3. 真机调试：通过微信开发者工具上传体验版，在真实设备测试性能
4. 性能分析：使用Three.js Stats.js性能监控 + 微信性能面板，优化内存与帧率
5. 分包配置：使用Webpack代码分割管理主包/分包，确保主包≤4MB
6. 资源上传：美术资源上传至腾讯云COS并配置CDN，生成资源清单
7. 提交审核：通过微信公众平台提交小游戏审核，准备相关资质材料
8. 发布上线：审核通过后全量发布，监控线上数据与用户反馈

## 4.6 第三方服务集成

• 微信登录：获取用户OpenID和基础信息，建立玩家账号体系
• 微信支付：内购道具和赛季通行证，支持多种支付方式
• 微信广告：激励视频和插屏广告变现，平衡用户体验与收益
• 腾讯云服务：COS存储、CDN加速、云函数（后端逻辑）
• 社交分享：微信好友/群分享，带参数二维码邀请
• 数据统计：微信数据助手 + 自定义事件埋点，精细化运营

# 5. 美术与音效规范

## 5.1 视觉风格

核心美术风格：Low-Poly（低多边形）+霓虹光效
色彩体系：主色调为深空紫、星云蓝，交互色为赛博朋克粉、荧光绿
UI设计：全息投影风格，半透明浮动面板

## 5.2 动画与特效

方块消除：粒子爆炸+空间扭曲特效
建筑升级：模块化生长动画
特殊技能：全屏特效，但不超过1.5秒

## 5.3 音效设计

背景音乐：氛围电子乐，每个星系有变奏
交互音效：未来感合成器音效
语音：AI导航员（可选多语言声线）

# 6. 商业化设计

## 6.1 货币体系

星币：关卡奖励、每日任务，购买基础道具、加速建筑，可付费
水晶：赛季通行证、成就，购买皮肤、特殊技能，可付费
贡献点：合作模式、社区贡献，兑换创意工坊专属物品，不可付费

## 6.2 付费点设计

小额度付费（占比70%）：星币补充包（¥6/¥30/¥98），月度探索卡（¥30，每日领取资源）
中额度付费（占比25%）：赛季高级通行证（¥68），星系扩展包（¥18-38/每个）
大额度付费（占比5%）：典藏版皮肤套装（¥128-328），创作者支持包（¥198，含专属标识）

## 6.3 广告变现（可选）

激励视频：额外复活机会、双倍资源奖励
插屏广告：仅限免费用户，每30分钟最多1次
无强制弹窗广告

# 8. 风险与应对

## 8.1 技术风险

三维操作在移动端体验不佳：提供"辅助视角"模式，自动高亮可操作方块
UGC内容审核压力：AI预审+社区举报+人工复核三级机制

## 8.2 市场风险

玩法过于复杂：渐进式引导，前5关只教基础消除
与《羊了个羊》同质化：强调三维、建设、叙事等差异化宣传

## 8.3 商业化风险

付费点过多影响口碑：所有付费均为外观或加速，不影响核心消除平衡
广告体验差：严格限制广告频率，提供付费去广告选项（¥30永久）
