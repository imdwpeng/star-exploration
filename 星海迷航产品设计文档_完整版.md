# 1. 产品概述

## 1.1 产品愿景

打造一款融合"三维消除"、"叙事探索"与"轻度社交"的下一代休闲手游，为玩家提供沉浸式的宇宙探险体验。

## 1.2 核心定位

目标用户：18-35岁休闲手游玩家，科幻爱好者，喜欢《羊了个羊》但渴望更深层次体验的用户
平台：iOS/Android/微信小程序（轻量版）/PC（Steam）
核心循环：消除→收集→建设→探索→社交分享

## 1.3 关键指标（KPI）

• 次日留存率：>40%
• 7日留存率：>20%
• 平均会话时长：>12分钟
• 付费转化率：3-5%
• 用户生成内容（UGC）关卡月活跃创作者：>1万人

# 2. 核心玩法设计

## 2.1 核心循环与唯一目标

单局游戏目标：清空三维星图上的所有方块。
失败条件：屏幕底部的"待消除槽"被填满，或者倒计时为0。

## 2.2 基础操作与核心规则（重中之重，必须明确）

规则1：消除发生的唯一地点
消除只发生在屏幕底部的待消除槽中。游戏场（三维星图）上的方块永远不会因为彼此相邻而自动消除。

规则2：消除触发的唯一方式
玩家必须手动点击场上的一个方块，将其移入待消除槽。当且仅当待消除槽内因这次放入而存在至少3个相同类型的方块时，这组方块会立即从槽中消除，并为玩家补充基础能量（防止失败的关键资源），增加相应倒计时。

规则3：场上图案与资源奖励的独立判断
"场上方块的排列图案"是一个独立的奖励系统，与"槽内能否消除"无必然关系。
每当玩家点击选择一个场上方块时，系统会同步进行两次独立判断：

判断1: 槽内兼容性判断：这个方块放入后，待消除槽内是否有3个同类的？

判断2: 场上奖励判断：这个方块在被移走前，在三维星图中是否构成了某个特定的"奖励图案"的一部分？

只有"判断1"直接影响消除和游戏进程。"判断2"只影响额外奖励资源的获取。

## 2.3 机制详解与示例

### 2.3.1 玩家单次操作触发逻辑流程图

在玩家点击场上方块X时，系统会同步进行两个独立判断：

1. 判断1（槽内状态）：方块X移入待消除槽，检查槽内是否有3个同色方块，如有则执行槽内消除并获得基础能量。
2. 判断2（场上图案）：检查方块X在被移走前是否属于某个奖励图案，如是则额外获得一份资源。

### 2.3.3 举例说明

场景：场上有三个红色方块水平相邻（构成奖励图案"晶簇"）。待消除槽中已有2个红色方块。

玩家操作：点击场上那个"三连红"中的中间方块。
系统判定：
1\. 槽内判断：该红色方块入槽，与槽内原有2个红色方块凑成3个，触发槽内消除。玩家获得基础能量。
2\. 场上判断：该方块在被移走前，属于场上"三个水平相邻红方块"图案。因此，玩家额外获得一份"晶体"资源。
结果：待消除槽清空一组红色方块，场上"三连红"被取走中间一个，剩下两个分开的红色方块。玩家获得"能量+晶体"。

# 3. 游戏模式

## 3.1 主线模式：星海编年史 [text](主线模式.md)

8个主题星系，每个星系5-8个关卡。每个星系有独特剧情、美术风格和专属机制。

## 3.2 合作探索（异步+实时）

异步合作：好友可派遣无人机构造你的太空站。
实时协作：3人小队挑战"深渊裂隙"特殊关卡，玩家各自负责一个维度的消除，共享资源池，共同决策建设选择。

## 3.3 社交裂变设计（微信小游戏核心）

### 3.3.1 分享裂变机制

**分享场景设计：**

1. **首次通关分享**：玩家首次通关关卡时，可选择分享到群/好友，获得"星图碎片"奖励（用于兑换稀有皮肤）
2. **求助分享**：倒计时剩余30秒且槽满度>80%时，可发送求助分享，好友点击后获得"能量补给包"（+50能量）
3. **排行榜挑战**：周排行榜前10名玩家可发起"好友挑战"，被挑战者点击后进入同关卡比拼，胜者获得双倍奖励
4. **助力解锁**：新玩家注册后可发起"新手助力"，好友助力后解锁专属皮肤"初航者"

**分享文案规范（微信合规）：**

- ✅ "帮帮我，一起探索星海！"
- ✅ "送你一个星系探索权限"
- ✅ "这个星系太难了，快来帮我！"
- ❌ "求你帮帮我"（诱导性用语）
- ❌ "下载游戏帮我"（诱导下载）

### 3.3.2 群排行系统

**群排行榜设计：**

1. **实时群排行**：玩家加入游戏群后，自动加入群排行，展示7日最高分
2. **群挑战活动**：每周五-周日开启群挑战，群总分达目标解锁全群奖励（星币/体力）
3. **群排名奖励**：
   - 第1-3名：专属群称号+稀有皮肤
   - 第4-10名：限定表情包+月卡
   - 第11-50名：星币补充包

**群分享转化漏斗优化：**

- 点击分享 → 群列表选择 → 群内卡片展示 → 点击进入 → 3秒加载完成 → 转化引导
- 关键优化点：卡片预览图展示"好友已玩"标识、实时群排名位置

### 3.3.3 助力与接力玩法

**新手助力：**

- 新玩家注册后自动触发"新手7日助力"
- 每日可发起1次助力，好友助力后获得"星图碎片"（7碎片兑换限定皮肤）
- 助力成功率>65%触发"助力达人"成就，奖励体力×30

**好友接力：**

- 关卡失败时可发起"好友接力"，好友点击后继承当前进度继续挑战
- 每关限用1次，每日最多3次
- 接力成功双方各得"探索者勋章"

### 3.3.4 社交炫耀系统

**成就分享：**

- 达成成就时自动生成分享卡片（含成就图标+进度条）
- 支持分享到朋友圈/群聊
- 例如："我已解锁'星系征服者'成就，排名前1%！"

**战绩分享：**

- 单局游戏结束后生成战绩卡片
- 包含：通关时间、消除次数、获得资源、排名
- 支持"晒战绩"按钮一键分享

**订阅消息：**

- 每日1次订阅消息（可关闭）
- 内容：昨日群排名、好友助力结果、活动提醒
- 示例："你的好友'太空探险家'助力了你，获得星图碎片×1"

## 3.4 创意工坊（UGC）

提供关卡编辑器，玩家可自定义方块初始布局、胜利/失败条件、自定义事件规则。发布后接受社区评分，高分关卡进入官方推荐。

## 3.4 赛季与活动

每季度一个主题赛季（如"虫族入侵"），赛季通行证（免费+付费），限时特殊关卡和排行榜。

## 3.5 新手引导与留存机制

### 3.5.1 新手引导设计（30秒上手）

**阶段1：核心玩法引导（0-5分钟）**

- 关卡1-3：纯教学关卡，强制引导操作
- 关卡4-5：自由尝试，背景音乐渐强
- 关卡6：正式关卡，引导结束

**引导要点：**

1. 第一次点击：高亮显示可操作方块
2. 第一次消除：慢动作播放+文字提示"3个相同即消除"
3. 第一次失败：弹出"求助分享"提示，赠送50能量
4. 第一次成功：展示"分享得奖励"按钮

- 屏幕顶部显示"探索进度"条（0%→100%）

## 4.3 关键算法与性能优化

• 三维布局生成算法：保证每关有至少3种可行解，算法复杂度O(n²)
• 动态资源加载算法：基于玩家进度预测加载CDN资源，减少等待时间
• 内存优化算法：自动检测并释放长时间未使用的资源，LRU缓存策略
• 启动优化：使用Webpack代码压缩和资源优化，首包体积减少60%
• 渲染优化：使用Three.js InstancedMesh批量渲染同类方块，减少Draw Call，目标移动端60fps
• 场景管理：使用Three.js Scene/Group层级管理，按需加载/卸载场景节点
• 动画优化：使用requestAnimationFrame驱动渲染循环，避免不必要的重绘

## 4.4 数据存储架构

• 玩家进度：本地加密存储（微信文件系统）+ 云端备份（可选腾讯云数据库）
• 游戏配置：远程JSON配置，支持热更新，无需重新发布小游戏
• UGC关卡：分布式存储（腾讯云COS）+ CDN加速，全球玩家快速访问
• 防作弊：关键操作服务器验证，本地数据签名校验，防止修改器
• 数据分析：集成微信数据助手，实时监控用户行为与付费转化

## 4.5 开发与发布流程

1. Three.js开发阶段：使用Three.js引擎，针对微信小游戏平台开发，注意避免使用微信不支持的API
2. 本地测试：VS Code + 微信开发者工具模拟器，验证基础功能

## 7.6 风险与应对

### 7.6.1 技术风险

**三维操作在移动端体验不佳：**

- 风险：玩家操作困难，流失率高
- 应对：提供"辅助视角"模式，自动高亮可操作方块；增加"一键旋转"功能；优化触摸交互

**UGC内容审核压力：**

- 风险：UGC内容质量参差不齐，审核成本高
- 应对：AI预审+社区举报+人工复核三级机制；设置内容规范；高分关卡自动通过

**性能问题：**

- 风险：低端机卡顿，闪退率高
- 应对：性能监控+自动降级；资源分包加载；动态调整画质

### 7.6.2 市场风险

**玩法过于复杂：**

- 风险：玩家上手困难，流失率高
- 应对：渐进式引导，前5关只教基础消除；提供"快速通关"模式；降低初期难度

**与《羊了个羊》同质化：**

- 风险：用户认为是抄袭，缺乏差异化
- 应对：强调三维、建设、叙事等差异化宣传；突出社交、UGC等微信生态优势

**用户增长放缓：**

- 风险：自然增长乏力，DAU停滞
- 应对：加强社交裂变；举办赛季活动；跨界联动

- 风险：收入不达预期，ROI低
- 应对：优化付费点触发时机；A/B测试价格与文案；提供新手期专属优惠
